직접 구현할 일은 거의 없으니, D3+React 시각화를 검증·리뷰·설계할 때 신경 써야 할 것 위주로 정리한다.

## 멘탈 모델 (리뷰의 전제)

D3+React는 역할이 나뉜다: **React가 빈 `<svg>` 틀과 상태(state)를 들고, D3는 `useEffect` 안에서 그 svg를 잡아 [데이터 → scale → 도형]으로 채운다.** 이 분담이 깨진 코드가 대부분 문제의 원인이라, 리뷰의 첫 질문은 "React가 관리하는 DOM과 D3가 만지는 DOM이 겹치지 않는가"다.

## 리뷰 체크포인트

- **React ↔ D3 경계**: D3는 `ref`로 잡은 svg **내부**만 조작해야 한다. React가 그리는 요소를 D3가 지우거나 바꾸면 React의 가상 DOM과 충돌해 요소가 사라지거나 깜빡인다. (React = 구조·state, D3 = svg 내부 그리기.)
- **useEffect 의존성 배열**: 버그의 단골. 바뀌는 state를 deps에 안 넣으면 그림이 안 갱신되고, 반대로 매 render마다 다시 그리면서 이전 요소를 안 지우면 도형이 중첩된다. deps가 실제 의존과 일치하는지, 재실행 때 정리(cleanup)를 하는지 확인.
- **Data join의 exit 처리**: 데이터가 갱신되는 시각화라면 `enter`만 있고 `exit().remove()`(또는 `.join`)가 없으면 오래된 도형이 안 지워지고 쌓인다. 정적(1회) 차트면 덜 중요.
- **Scale의 domain·방향**: `domain`이 데이터 범위와 안 맞으면(하드코딩 등) 값이 잘리거나 뭉친다 — 보통 `d3.extent(data)`처럼 데이터에서 뽑는지 확인. y축은 `range([height, 0])`로 뒤집었는지(안 하면 상하 반전).
- **이벤트 핸들러의 this**: `this`(이벤트가 난 element)를 쓰는 핸들러는 화살표 함수로 쓰면 깨진다 → 일반 `function`인지 확인.
- **성능**: SVG 도형이 수천 개면 느려진다 → 대량이면 canvas/WebGL을 고려. brushing·transition 안에 무거운 계산이 있으면 프레임마다 돌아 버벅인다(무거운 건 start·end 시점에만).
- **버전**: D3는 메이저 버전마다 API가 다르다(특히 `.on`·join). 예제와 코드의 버전이 섞이면 안 도니 버전부터 확인.

## 설계 시 고려

- **역할 분리**: 뷰별로 React 컴포넌트를 나누되, 여러 뷰가 연동(linking)되면 공유 state를 상위 컴포넌트로 올려 자식들이 같은 선택을 함께 구독하게 한다.
- **반응형**: 크기를 고정 픽셀로 박으면 화면 변화에 안 맞는다 → 컨테이너 크기를 재서 scale의 range에 반영(리사이즈 시 다시 그림).
- **읽기 좋은 그림**: 색맹 안전 팔레트, 축·라벨·단위 명시, 데이터가 아닌 장식(잉크) 최소화. "어떤 데이터에 어떤 차트"는 EDA 판단.

## 프롬프트에 넣으면 좋은 지시 (AI로 만들 때)

위 문제들을 애초에 막도록, 생성 프롬프트에 이런 요구를 박아두면 좋다.

- "D3는 `useRef`로 잡은 `<svg>` **내부에서만** 조작하고, React가 렌더링하는 DOM은 건드리지 말 것" — 경계 충돌 방지.
- "data join은 `.join()`으로 처리해 데이터 갱신 시 **exit(오래된 요소 제거)** 까지 되게 할 것."
- "`useEffect` 의존성 배열에 갱신 트리거 state를 정확히 넣고, 재실행 시 이전 그림을 **정리(cleanup)** 할 것."
- "scale의 domain은 `d3.extent` 등으로 **데이터에서 계산**(하드코딩 금지), y축은 뒤집을 것."
- "**D3 v7 기준**으로 작성" — 버전 혼재로 API가 어긋나는 것 방지.
- "차트 크기는 컨테이너에 맞춘 **반응형**으로(고정 픽셀 지양), 리사이즈 시 다시 그릴 것."
- "**색맹 안전 팔레트**, 축·라벨·단위·범례 포함, 접근성(aria-label)·툴팁 제공."
- "데이터가 수천 개 이상이면 SVG 대신 **canvas**로."
- "이벤트 핸들러에서 `this`를 쓰면 화살표 함수 대신 일반 `function`으로."
