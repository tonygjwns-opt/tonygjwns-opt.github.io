최적화·집계 문제의 세 도구 — DP는 부분문제를 저장해 재사용, 그리디는 매 순간 국소 최적, 누적합은 전처리로 구간 질의를 O(1)에.

## DP (동적계획법)

본질: **점화식 + 초기조건**. 부분문제의 답을 저장해 재사용한다.

- 상태 정의가 절반이다. 단순 점화로 안 되면 직전 정보(연속 개수 등)를 상태에 추가한다.
- 초기조건·경계(n=1,2)를 먼저 점검. 큰 수는 modulo로 줄여 저장.
- 대표 틀:
  - 연속합(Kadane): `b[i] = a[i] + max(b[i-1], 0)`
  - 경로: `b[i][상태] = a[i][상태] + (이전 행에서 갈 수 있는 상태의 min/max)`
  - LCS: 같으면 대각+1, 다르면 max(위, 왼)
  - LIS: `dp[i] = (앞쪽 작은 값 중 dp[j] 최대) + 1`, 답은 max(dp). 바이토닉은 정/역방향 LIS 합 −1.

```python
def max_subarray(a):           # 연속 부분합 최대 (Kadane)
    best = cur = a[0]
    for x in a[1:]:
        cur = x + max(cur, 0)  # 직전까지 이어붙일지 vs 여기서 새로 시작할지
        best = max(best, cur)
    return best
```

## 그리디 (greedy)

본질: 매 순간 국소 최적을 골라 전체 최적에 도달. 단 **'왜 항상 최적인가'를 교환 논증으로 정당화해야** 성립한다.

- 정렬 기준을 바꾸면 대부분 풀린다(끝나는 시간 순 = 구간 스케줄링, 짧은 작업 순 = 합 최소화).
- 진행하며 지금까지의 최선(최저가 등)을 들고 가는 패턴.
- 안 통하는 구조(예: 동전 액면이 배수 관계가 아님)면 DP로 간다.

## 누적합 (prefix sum)

본질: 미리 더해두면 임의 구간 합을 O(1)에. **반복 질의를 전처리로 바꾸는** 발상.

- `cum[i] = cum[i-1] + a[i]`, 구간 [l, r] 합 = `cum[r] - cum[l-1]`.
- 고정 길이 구간은 더하고 빼며 미끄러뜨린다(슬라이딩 윈도우): `t[i+1] = t[i] + a[i+k] - a[i]`.
- 카테고리별로 따로 누적하면 '구간 내 특정 값 개수'도 O(1).
- 누적합 % M이 같은 두 지점 사이 합은 M의 배수 → 나머지별 개수로 nC2.
- 2차원도 같은 원리로 직사각형 합을 O(1)에.

```python
from itertools import accumulate
cum = [0] + list(accumulate(a))   # cum[i] = a[0..i-1] 합
# 구간 [l, r] (0-index, r 포함) 합 = cum[r+1] - cum[l]
```
