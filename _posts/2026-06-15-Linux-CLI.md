## 개념

- **CLI**(Command Line Interface): 명령어로 파일·폴더를 찾고 만드는 인터페이스(GUI 반대).
- **shell**: CLI의 일종(명령 해석기). `bash`(Bourne-Again Shell)는 리눅스·macOS 기본 shell.
- 윈도우에선 보통 **git bash**를 써서 bash 환경을 사용.

## 탐색·이동

- `pwd` : 현재 위치 확인. (※ 윈도우 cmd는 `cd`가 현재 위치 표시, `dir`이 `ls` 역할.)
- `cd 디렉토리` : 이동. 상위로는 `cd ..`.
- `ls` : 파일 목록.
  - `-a` 숨김 포함 / `-l` long(테이블) / `-t` 수정시간순. 조합 가능(`ls -alt`).
  - `-l` 컬럼 순서: 접근권한 / 하드링크 수 / 소유자 / 소유 그룹 / 크기(byte) / 수정시간 / 이름.
- `find / -name '*.apk'` : 이름 패턴으로 찾기.

## 파일 보기·편집

- `cat a.txt` : 파일 내용 출력.
- `vi a.c` : 편집기로 열기/생성. `i`=insert, 저장·종료는 `ESC` → `:wq`.

## 파일·디렉토리 조작

- `mkdir test` : 디렉토리 생성.
- `cp source dest` : 복사. `cp src dir/`, `cp f1 f2 dir/`, `cp * dir/`, `cp w*.txt dir/`(w로 시작 .txt만).
- `mv` : 이동(사용법 `cp`와 동일). `mv old.txt new.txt` : 이름 변경.
- `rm 파일` : 삭제. `rm -r dir` : 디렉토리 + 하위까지.
  - 옵션: `-i`(하나씩 확인) / `-f`(강제, 없는 파일 무시·확인 생략) / `-rf`(디렉토리 강제 재귀 — ★위험, 복구 불가). `rmdir`(빈 디렉토리만).
  - ※ `rm`은 휴지통을 안 거치고 즉시 영구 삭제 → 와일드카드(`rm *`) 특히 조심.
- **휴지통 비우기**: `rm -rf ~/.local/share/Trash/*` (rm은 휴지통을 안 거치므로 휴지통은 따로 비움.)

## 리다이렉션 / 파이프

- `echo "Hello" > hello.txt` : 출력을 파일에(덮어쓰기). `>>` 는 append.
- `cat a.txt > b.txt` / `cat a.txt >> b.txt` : 덮어쓰기 / 이어붙이기.
- `cat < a.txt` : a 내용을 입력으로.
- `|`(파이프) : 왼쪽 출력 → 오른쪽 입력. 예: `cat a.txt | wc | cat > b.txt`.

## 텍스트 처리

- `wc` : 줄/단어/글자 수.
- `sort a.txt` : 정렬. `uniq a.txt` : 중복 제거(**인접한 것만**! 보통 `sort | uniq`).
- `grep America file` : 포함 줄 출력(대소문자 구분). `-i` 무시 / `-R` 디렉토리 전체 / `-Rl` 파일명만. 예 `grep -R player .`.
- `sed 's/snow/rain/' file` : 찾아 교체(각 줄 처음 것만, 끝에 `g`면 전부). `s`=교체, `snow`=대상, `rain`=교체값.

## tmux (터미널 멀티플렉서)

용도: 한 터미널에서 여러 창·분할 + **세션을 백그라운드로 유지**(SSH가 끊겨도 안의 프로세스 계속 돎).

**세션**

```bash
tmux                       # 새 세션 시작
tmux new -s 이름           # 이름 지정해 시작
# Ctrl+b  d                : detach(세션은 살아있고 빠져나옴)
tmux ls                    # 세션 목록
tmux attach -t 이름        # 다시 붙기
tmux kill-session -t 이름  # 세션 종료
```

**창·분할** (먼저 prefix `Ctrl+b`를 누른 뒤 키)

- `c` 새 창 / `n`,`p` 창 이동 / `,` 창 이름변경
- `%` 좌우 분할 / `"` 상하 분할 / `화살표` pane 이동 / `x` pane 닫기

> tmux의 모든 단축키는 prefix(`Ctrl+b`)를 먼저 누른 다음 입력한다.

## 컴파일·실행 (C/C++)

- `gcc hello.cpp -o hello` : 컴파일(`-o`로 이름, 생략 시 `a.out`). 파일이 현재 위치에 있어야 함.
- `gcc -Wall` : 모든 경고. `./hello` : 실행.
- `man strlen` : 함수 설명 확인.

## argc / argv (실행 인자)

- `argc` = `argv`의 크기. `argv[0]`에 실행 명령이 들어가므로 `argc`는 항상 1 이상.
- `./argstest x y` → `argv[1]=x`, `argv[2]=y`
- `./argstest "x y"` → `argv[1]="x y"` (따옴표로 묶으면 하나의 인자)
