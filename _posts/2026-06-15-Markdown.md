## 본질

- **Markdown**: text-to-HTML. plain text로 쓰면 HTML로 변환·렌더된다. HTML보다 쉽고 가볍다.
- plain text라 메모장으로도 작성 가능. `.md` 확장자 권장(`.txt`로도 되긴 함). 변환은 별도 앱/뷰어가 HTML로 렌더.
- **렌더링해서 보기**: 메모장은 raw(기호 그대로)만 보인다. 제대로 보려면 — VS Code(미리보기 `Ctrl+Shift+V`, 옆창 `Ctrl+K V`), Typora, Obsidian, GitHub(`.md` 자동 렌더), 온라인(StackEdit·Dillinger).
- 쓰는 이유: 접근 쉬움, 무거운 에디터 불필요, 거의 모든 곳(특히 오픈소스·GitHub)에서 쓰임.
- 쓰이는 곳: 웹사이트, 이메일, documentation.
- ※ 확장 버전이 여럿(예: GitHub Flavored Markdown)이라 쓰는 환경의 문법을 확인할 것.
- ※ **부작용**: 워낙 널리 쓰여서 LLM 채팅·각종 입력창이 마크다운을 자동 렌더링함 → `~취소선~`·`**볼드**`·`#` 같은 기호가 의도와 다르게 변환돼 은근히 방해되기도(그래서 raw text를 선호하기도).

## 주요 문법

- 제목(Heading): `# 제목` (`#` 1~6개, 많을수록 레벨 높고 작아짐). 또는 글 아랫줄에 `=====`(h1).
- 강조: `*기울임*`, `**볼드**`, `***볼드+기울임***`, `~취소선~`.
- 목록: `*` `+` `-` (모두 불릿 점, 한 종류로 통일 권장). `1.` `2.`는 번호 목록(자동 들여쓰기).
- 체크박스(할일 목록, GitHub): `- [ ]` 미완 / `- [x]` 완료.
- 인용: `>` (옆에 줄). 중첩 목록은 한 줄 띄고 4칸 들여쓰기.
- 코드: `` `짧은 인라인` ``. 백틱 3개로 감싸면 여러 줄 박스 — 언어를 적으면(` ```python `) 문법 강조(syntax highlight).
- 표(table):

  ```
  | 헤더 | 헤더 |
  |---|---|
  | 셀  | 셀  |
  ```

  구분선 `|---|`의 콜론으로 정렬: `:---` 좌 / `:---:` 중앙 / `---:` 우.
- 수평선: `---` 또는 `***` (구분선).
- 링크: `[내용](URL)`, 이미지: `![대체텍스트](파일/URL)`.
- 첨자: `<sub>아래첨자</sub>` / `<sup>위첨자</sup>` (HTML 태그를 그대로 써도 먹힌다).
- 각주: 본문에 `[^1]`, 문서 어딘가에 `[^1]: 설명`.
- 이스케이프: `\*` `\#` `` \` `` 처럼 앞에 `\` → 기호를 문법이 아니라 '글자 그대로' 표시.

> **줄바꿈 규칙(중요)**: 마크다운은 '한 줄 띄움'(single newline)을 무시해 문장을 이어붙인다. 줄을 바꾸려면 **줄 끝에 공백 2칸**, 문단을 나누려면 **빈 줄 1개**. (raw text 쓰던 사람이 가장 많이 당하는 부분.)

## README 국룰

- README는 보통 **마크다운(`README.md`)** 으로 작성 → GitHub repo 첫 화면에 자동 렌더되어 보인다.
- README는 꼭 쓸 것. 프로젝트에 시간을 많이 썼으면 README에도 시간을 들인다(간단하되 상세히).
- 넣을 요소: 프로젝트 이름 / 설명 / feature / how to use / 기술 스택 / 협력자 / 라이선스.
- 헤더로 구획을 나누고, 사진은 `![](경로)`.
