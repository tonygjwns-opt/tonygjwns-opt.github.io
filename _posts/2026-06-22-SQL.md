relational DB는 정보를 하나 이상의 표(table)로 관리한다. 각 표는 행(row=레코드)과 열(column=속성)으로 되어 있고, SQL로 표를 만들고 조회·편집한다. 문장 끝은 `;`, 명령어는 관례상 대문자(대소문자 구분은 안 함). RDBMS는 PostgreSQL·MySQL·SQLite 등 — SQLite는 DB가 단일 파일이라 서버 없이 가볍게 쓰지만 동시접근에 약하다. 퀀트는 데이터를 DB에서 꺼내오는 일이 잦아 SELECT·JOIN·GROUP BY가 특히 중요하다.

## 조회 (SELECT)

`SELECT`는 "어떤 열을", `FROM`은 "어떤 표에서" 가져올지 지정한다. 결과는 원본을 건드리지 않고 result set이라는 새 표로 반환된다.

```sql
SELECT * FROM movies;                 -- * = 모든 열
SELECT name, year FROM movies;        -- 특정 열만
SELECT DISTINCT genre FROM movies;    -- 중복 제거하고 고유값만
SELECT name AS 'Title' FROM movies;   -- 결과에서 열 이름을 Title로 표시(별칭)
```

## 조건 (WHERE)

WHERE는 조건을 만족하는 행만 남긴다. 비교 연산자는 대체로 프로그래밍과 같지만 표기가 조금 다르다: 같음은 `=` 하나(`==` 아님), 다름은 `!=`(또는 `<>`).

```sql
SELECT * FROM movies WHERE rating > 8;   -- 초과
WHERE genre =  'comedy';                  -- 같음은 = 하나
WHERE genre != 'comedy';                  -- 다름은 != (또는 <>)
WHERE year BETWEEN 2000 AND 2010;         -- 2000 이상 2010 이하(양끝 포함)
WHERE rating > 8 AND genre = 'comedy';    -- 조건 결합 AND / OR
```

`=` · `IS` · `LIKE` 구분:

- **`=`**: 값이 정확히 같은지. 문자열도 정확히 일치면 `WHERE name = 'Alice'`.
- **`LIKE`는 '패턴' 매칭 전용** — 부분 일치·와일드카드가 필요할 때만. `_`(아무 한 글자), `%`(아무 길이 문자열). 예: `'A%'`(A로 시작), `'%man%'`(man 포함), `'se_en'`(seven 등). 정확히 일치만 보면 되면 `=`로 충분.
- **`IS`는 NULL 검사 전용**. NULL(값 없음)은 `= NULL`로 비교하면 항상 "알 수 없음"이라 안 걸린다 → `IS NULL` / `IS NOT NULL`. (파이썬에서 값은 `==`, None은 `is`로 보는 것과 비슷.)

BETWEEN은 텍스트에도 되는데 이때 **사전순** 비교라 함정이 있다: `BETWEEN 'A' AND 'J'`는 `'J'` 자체는 포함하지만 `'James'`는 미포함이다(사전순에서 짧은 `'J'`가 `'Ja...'`보다 앞서 `'James'`가 `'J'`를 넘어감). J로 시작하는 이름까지 넣으려면 `< 'K'`로 잡는다.

## 정렬·제한

```sql
SELECT * FROM movies ORDER BY year DESC LIMIT 10;
-- ORDER BY: 정렬(DESC 내림 / ASC 오름). LIMIT: 상위 N행만.
```

실행 순서상 WHERE(필터) → ORDER BY(정렬) → LIMIT(자르기).

## 집계 (GROUP BY)

집계 함수는 여러 행을 하나의 값으로 압축한다: `COUNT`(개수)·`SUM`·`AVG`·`MAX`·`MIN`.

```sql
SELECT AVG(price), MAX(price), COUNT(*) FROM inventory;   -- 표 전체를 한 줄로 요약
```

`GROUP BY`를 붙이면 "같은 값끼리 묶어 그룹마다 집계"한다. movies를 genre로 묶으면 각 장르가 한 행이 되고 COUNT는 그 장르의 행 수가 된다:

```
movies                    SELECT genre, COUNT(*) FROM movies GROUP BY genre
name  genre               genre    COUNT(*)
A     comedy              comedy   2
B     drama       →       drama    1
C     comedy
```

집계 결과를 다시 조건으로 거를 땐 `WHERE`가 아니라 `HAVING`을 쓴다(WHERE는 묶기 '전' 행 단위 필터라 집계값을 못 봄):

```sql
SELECT genre, COUNT(*) FROM movies
GROUP BY genre HAVING COUNT(*) > 5;   -- 5편 넘는 장르만
```

팁: `GROUP BY 1` / `ORDER BY 1`은 SELECT에 나열한 첫 번째 열을 가리킨다(긴 식을 다시 안 써도 됨).

## 결합 (JOIN)

먼저 두 용어:

- **PK(primary key, 기본키)**: 각 행을 유일하게 식별하는 열(고유·NULL 불가). 예: customers 표의 `cid`.
- **FK(foreign key, 외래키)**: 다른 표의 PK를 가리키는 열(중복 가능). 예: orders 표의 `cid`는 "어느 고객의 주문인지"를 customers.cid로 가리킨다. FK가 두 표를 잇는 고리다.

JOIN은 두 표를 이 고리(보통 FK=PK)로 옆으로 이어 붙인다. 예시:

```
orders            customers
oid  cid          cid  name
1    10           10   Alice
2    10           20   Bob
3    99           30   Carol
```

**INNER JOIN**(그냥 `JOIN`) — 조건이 맞는 쌍만 남긴다(교집합):

```sql
SELECT oid, name FROM orders JOIN customers ON orders.cid = customers.cid;
-- 각 orders 행에 대해 cid가 같은 customers 행을 찾아 옆에 붙임. 못 찾으면 버림.
--   → (1, Alice), (2, Alice)
--   oid 3은 cid 99가 customers에 없어 빠지고, Carol(30)은 주문이 없어 빠짐.
```

**LEFT JOIN** — 왼쪽(orders) 행은 전부 유지, 짝이 없으면 오른쪽 열은 NULL:

```sql
SELECT oid, name FROM orders LEFT JOIN customers ON orders.cid = customers.cid;
--   → (1, Alice), (2, Alice), (3, NULL)
```

- **CROSS JOIN**: 두 표의 모든 조합(곱집합, 3×3 = 9행). ON이 없다.
- **UNION**: JOIN이 옆으로 붙이는 것과 달리, 두 SELECT 결과를 **위아래로** 이어 붙인다(열·타입이 같아야). 기본은 중복 제거, `UNION ALL`은 중복 유지.

## CASE (행별 조건 분기)

행마다 값에 따라 다른 결과를 내는 새 열을 만든다(프로그래밍의 if-elif-else).

```sql
SELECT name,
  CASE WHEN rating > 8 THEN 'good'
       WHEN rating > 6 THEN 'ok'
       ELSE 'bad' END AS review     -- END AS 로 열 이름 지정(안 하면 CASE문 전체가 열 이름이 됨)
FROM movies;
```

## CTE (WITH: 이름 붙인 임시 결과)

`WITH 이름 AS (쿼리)`는 그 쿼리 결과에 이름을 붙여, 아래 본 쿼리에서 하나의 표처럼 쓰게 해준다. "그 쿼리 동안만 존재하는 임시 표"라고 보면 된다.

```sql
WITH top AS (SELECT * FROM movies WHERE rating > 8)   -- rating 8 넘는 영화만 담은 임시 표 'top'
SELECT genre, COUNT(*) FROM top GROUP BY genre;        -- 그 top을 대상으로 장르별 개수
```

즉 "먼저 rating>8인 영화만 뽑아 `top`이라 부르고, 그다음 그 top을 장르별로 세라"는 2단계다. CTE 없이 서브쿼리로 중첩해도 되지만(`FROM (SELECT * FROM movies WHERE rating > 8) GROUP BY genre`), WITH로 이름을 붙이면 단계가 눈에 보이고 재사용도 쉽다.

## 스칼라 함수 (행별 변환)

집계와 반대로, 행 하나하나를 변환한다.

```sql
SELECT strftime('%Y', ts), ROUND(AVG(price), 2) FROM t;
-- strftime: 날짜에서 부분 추출. %Y년 %m월 %d일 %H시 %M분 (소문자 m=월, 대문자 M=분)
-- ROUND(x, 2): 소수 2자리 반올림. 집계와 중첩 가능.
```

## 표 생성·편집

```sql
CREATE TABLE celebs (
  id    INTEGER PRIMARY KEY,   -- 기본키: 표당 하나, 자동으로 고유 + NOT NULL
  name  TEXT UNIQUE,           -- 값이 서로 달라야 함(NULL은 여러 번 허용)
  birth TEXT NOT NULL,         -- NULL 금지
  note  TEXT DEFAULT 'N/A'     -- 값 안 주면 기본값
);
INSERT INTO celebs (id, name) VALUES (1, 'Justin');   -- 행 추가
ALTER TABLE celebs ADD COLUMN handle TEXT;            -- 'handle'이라는 TEXT 열을 새로 추가(기존 행은 그 값이 NULL)
UPDATE celebs SET handle = '@x' WHERE id = 1;         -- id=1 행의 handle을 변경
DELETE FROM celebs WHERE handle IS NULL;              -- handle이 NULL인 행 삭제
```

※ UPDATE·DELETE의 `WHERE`는 **선택**이다. 붙이면 조건에 맞는 행에만 적용되지만, 빼면 조건이 없어 **표의 모든 행에 적용된다** — `DELETE FROM celebs`는 전 행 삭제, `UPDATE celebs SET handle = '@x'`는 전 행의 handle을 바꾼다. 실수로 WHERE를 빠뜨리면 표가 통째로 날아가니 주의.
