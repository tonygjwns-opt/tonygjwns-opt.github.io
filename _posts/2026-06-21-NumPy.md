파이썬 리스트로 수치 계산을 하면 느리고 장황하다. NumPy는 "같은 타입 값이 연속 메모리에 박힌 배열(ndarray)"을 다뤄, 반복문 없이 배열 전체를 한 줄로 계산한다(벡터화). 퀀트 코드의 수익률·공분산·시뮬레이션이 거의 다 이 위에서 돈다. 멘탈모델 셋만 잡으면 된다.

- shape와 axis: 배열의 모양(행, 열, ...). 집계는 "어느 축으로 접을지"가 핵심.
- 연산은 요소별 + 브로드캐스트: 모양이 맞거나 한쪽이 1이면 자동 확장해 칸끼리 계산.
- 슬라이스는 view(원본 공유): 잘라낸 걸 고치면 원본도 바뀐다 → 독립본은 `.copy()`.

## 핵심: ndarray와 리스트의 차이

```python
import numpy as np
a = np.array([[1, 2, 3], [4, 5, 6]])
a.shape   # (2, 3)  각 축 크기
a.ndim    # 2       차원 수(rank)
a.size    # 6       총 원소 수
a.dtype   # int64   모든 원소가 같은 타입
```

리스트와 달리 크기 고정·동일 타입이라 연속 메모리에 놓여 대규모 수학에 훨씬 빠르다. `c = a * b` 한 줄로 요소별 곱(반복문 불필요).

## 생성

```python
np.array([1, 2, 3])
np.zeros((2, 3))         # 0. 으로
np.ones(4)               # 1. 으로
np.empty(3)              # 초기화 안 함(쓰레기값) — 곧 덮어쓸 때만
np.full((2, 2), 7)       # 7 로 채움
np.eye(3)                # 단위행렬
np.arange(0, 10, 2)      # [0 2 4 6 8]  (start, stop, step)
np.linspace(0, 10, 5)    # [0 2.5 5 7.5 10]  양끝 포함, 등간격 5개
```

구간을 step으로 끊으면 `arange`, 개수로 n등분하면 `linspace`.

## 모양 바꾸기

```python
a.reshape(3, 2)          # 모양 재배치(원소 수는 같아야)
a.reshape(-1)            # 1차원으로 펴기 (-1 = 알아서 계산)
a.T                      # 전치(transpose)
a[np.newaxis, :]         # (6,) → (1, 6) 축 추가. np.expand_dims(a, 0) 도 동일
np.concatenate([a, b], axis=0)   # 축 따라 이어붙이기
np.vstack([a, b])        # 세로로 쌓기
np.hstack([a, b])        # 가로로 붙이기
a.flatten()              # 1차원 새 배열(복사)
a.ravel()                # 1차원 view(원본 공유 — 고치면 원본도 바뀜)
```

## 인덱싱·슬라이싱·뷰 (★ 원본 공유 주의)

```python
s = a[1:3]               # 슬라이스는 view — 추가 메모리 없이 원본을 들여다봄
s[0] = 99                # → 원본 a 도 바뀜!  독립본이 필요하면 a[1:3].copy()

a[a < 5]                 # 불리언 인덱싱: 조건 맞는 값만
a[(a > 2) & (a < 11)]    # 여러 조건은 () 로 묶고 & | (and/or 아님)
a[a < 0] = 0             # 조건 위치에 일괄 대입 (순수 list 는 안 됨)
```

팬시 인덱싱(배열을 인덱스로):

```python
idx = np.array([0, 2, 0, 1])
a[np.arange(4), idx]     # 각 행에서 idx 가 가리키는 열을 뽑음
```

## 연산: 요소별·브로드캐스트·ufunc

```python
a + b, a * b, a ** 2     # 칸끼리 요소별
a + 1                    # 스칼라는 모든 칸에 (브로드캐스트)
np.array([[1, 2], [3, 4]]) + np.array([1, 2])   # (2,2)+(2,) → 행마다 [1,2] 더함
```

브로드캐스트 규칙: 뒤 축부터 맞춰보며 크기가 같거나 한쪽이 1이면 확장한다. 안 맞으면 에러.

```python
np.abs(a), np.sqrt(a), np.exp(a), np.log(a)   # ufunc: 요소별 수학 함수
np.isnan(a), np.isinf(a)                       # 결측/무한 판정(전처리에 자주)
```

## 집계·통계 (axis가 핵심)

```python
a.sum()                  # 전체 합
a.sum(axis=0)            # 행 방향으로 접음 = 열별 합
a.sum(axis=1)            # 열 방향으로 접음 = 행별 합
a.mean(), a.max(), a.min()
np.median(a), np.var(a), np.std(a)
np.std(a, ddof=1)        # ★ 표본 표준편차(n-1). 기본 ddof=0 은 모집단(n) 기준
np.quantile(a, 0.75)     # 3사분위. [.25,.5,.75] 배열도 가능 (percentile 은 0~100 스케일)
np.argmax(a), np.argmin(a)   # 최대/최소 '위치'(인덱스)
np.cumsum(a), np.cumprod(a)  # 누적합·누적곱 (수익률 누적에 자주)
np.corrcoef(x, y)[0, 1]      # 상관계수 r
```

axis 직관: "그 축을 따라 접는다(없앤다)". (2,3)에서 `axis=0`이면 결과 (3,), `axis=1`이면 (2,).

## 난수 (시뮬레이션)

```python
rng = np.random.default_rng(0)        # 시드 고정(재현 가능) — 최신 권장 방식
rng.random(3)                         # 0~1 실수 3개
rng.integers(0, 5, size=(2, 4))       # 0~4 정수 (2,4) 배열
rng.normal(0, 1, 1000)                # 정규분포 표본(몬테카를로)
rng.choice(['H', 'T'], size=10, p=[0.5, 0.5])   # 지정 확률로 샘플링
```

## 자주 쓰는 보조

```python
np.where(cond, x, y)               # 조건 참이면 x, 아니면 y (요소별 선택)
np.nonzero(a < 5)                  # 조건 만족 위치 인덱스
np.unique(a, return_counts=True)   # 고유값 (+ 등장 횟수)
x, y = np.meshgrid(x, y)           # 1D 둘 → 2D 격자(등고선·surface 전처리)
from scipy import stats            # 회귀·검정은 scipy
r = stats.linregress(x, y)         # r.slope, r.intercept, r.rvalue, r.pvalue
```
